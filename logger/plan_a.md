# Research Journal System — Implementation Plan

## Context

We want a persistent research journal that captures the narrative thread across work sessions — what was tried, what worked, what failed, decisions made, and next steps. The system should work in any git repo, not just ml-debate-lab. A separate "monitor" session can later synthesize the journal into a complete research narrative.

The system has three layers:
1. **Skills** — `/journal-entry` (manual) and `/research-note` (synthesis)
2. **Hooks** — SessionStart (load context), PostCompact (auto-capture), Stop (throttled reminder)
3. **Artifact** — `RESEARCH_JOURNAL.md` at repo root (per-project, append-only)

User decisions: direct-write for PostCompact (no staging file), user-level hooks (apply to all projects).

---

## Files to Create

### 1. `~/.claude/skills/journal-entry/SKILL.md`

User-invocable skill. Steps:
- `git rev-parse --show-toplevel` to find repo root
- Check if `RESEARCH_JOURNAL.md` exists; create with header if not
- Find last entry timestamp, run `git log --since` for recent changes
- Glob for common tracking files (`**/ISSUE*`, `**/lesson*`, `**/POST_MORTEM*`, `**/TODO*`) modified since last entry — discover, don't hardcode
- Synthesize structured entry from conversation context + git evidence:
  ```
  ## YYYY-MM-DD HH:MM — [one-line focus]
  **Goal:** ...
  **Work done:** ...
  **What worked:** ...
  **What didn't / surprises:** ...
  **Decisions:** ...
  **Open questions:** ...
  **Next:** ...
  ```
- Show draft, confirm before appending
- Append with `---` separator

### 2. `~/.claude/skills/research-note/SKILL.md`

User-invocable skill. Steps:
- Find repo root, read full `RESEARCH_JOURNAL.md` (stop if missing)
- Read `git log --oneline --all` for project timeline
- Discover supplementary .md files at repo root + 2 levels deep (issue trackers, lessons, post-mortems — match by filename patterns, not hardcoded paths)
- Synthesize into `RESEARCH_NARRATIVE.md` with sections: Problem Statement, Timeline, What Was Tried, What Failed, What Worked, Key Decisions, Current State, Open Questions
- Show draft, confirm before writing (overwrites — git has history)

### 3. `~/.claude/hooks/journal-session-start.sh`

Shell script. On SessionStart:
- Find repo root, check for `RESEARCH_JOURNAL.md`
- Extract last entry (from final `---` to EOF) using awk, truncate to 60 lines
- Emit JSON with `additionalContext` injecting latest entry
- Exit silently if no repo or no journal

### 4. `~/.claude/hooks/journal-postcompact.sh`

Shell script. On PostCompact:
- Find repo root
- Extract compaction summary from stdin JSON via `jq`
- Create `RESEARCH_JOURNAL.md` with header if it doesn't exist
- Append auto-entry directly (no confirmation — context is being lost):
  ```
  ## YYYY-MM-DD HH:MM — [auto: post-compaction snapshot]
  [compaction summary content]
  > Auto-generated by PostCompact hook. Use /journal-entry for a full structured entry.
  ```

### 5. `~/.claude/hooks/journal-stop-reminder.sh`

Shell script. On Stop (throttled):
- Read `session_id` from stdin JSON
- Increment counter in `/tmp/claude-journal-turns-${session_id}.txt`
- Every 15 turns, emit `additionalContext` reminder to update journal
- Exit silently otherwise

## Implementation Order

1. Create hook scripts (3 files, parallel) — make all executable with `chmod +x`
2. Create skill files (2 files, parallel with step 1)
3. Modify `~/.claude/settings.json` — add hooks key, preserve all existing content
4. Verify: `jq -e '.hooks' ~/.claude/settings.json` to confirm valid JSON
5. Test SessionStart hook: `echo '{}' | bash ~/.claude/hooks/journal-session-start.sh` in ml-debate-lab (should exit silently — no journal yet)
6. Test Stop hook: `echo '{"session_id":"test"}' | bash ~/.claude/hooks/journal-stop-reminder.sh` (should exit silently on first call)

---

## Dependencies

- `jq` — all hooks use it for JSON parsing/emission (already installed, used by statusline script)
- `git` — all components use `git rev-parse --show-toplevel` for repo detection
- No project-specific paths anywhere — fully portable across git repos

---

## Edge Cases

- **Not in a git repo:** All hooks and skills detect this and exit/stop silently
- **No RESEARCH_JOURNAL.md yet:** Skills create it on first use; SessionStart hook exits silently; PostCompact creates it if needed
- **Concurrent sessions:** Append operations are atomic under PIPE_BUF (4096 bytes) — entries are small enough
- **Auto entries vs manual:** Auto entries are clearly labeled `[auto: post-compaction snapshot]` so `/research-note` can weight them appropriately
