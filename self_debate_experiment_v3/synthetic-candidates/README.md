# synthetic-candidates/

This directory contains raw candidate benchmark cases generated by an external LLM as inputs to the v3 experiment pipeline.

## Generation

Cases were generated using the companion case generation prompt (`self_debate_experiment_v3/benchmark_case_generation_prompt.md`) executed on the **Perplexity Computer** framework with **GPT-5.4**. GPT-5.4 is the authoring model — it both designs the ML evaluation scenarios and writes the full case JSON, including task_prompt, planted_issues, scoring_targets, and ground_truth fields. Perplexity Computer orchestrates the session but the case content itself is GPT-5.4 output.

## Role in the pipeline

These files are raw LLM output — unvalidated and unverified. They feed into the v3 pipeline starting at Phase 0:

1. **Phase 0** — `validate_cases.py` checks schema compliance and composition targets (category balance, difficulty distribution, defense_wins coverage, must_find population)
2. **Phase 1** — `CASE_VERIFIER` agent reviews each case for flaw authenticity, benchmark quality, and external sourcing of defense_wins cases
3. Output of verification is `benchmark_cases_verified.json`, which is what the experiment actually runs against

The `.json` files in this directory are candidates only. Do not use them directly for scoring or analysis — use `benchmark_cases_verified.json` produced by Phase 1.

## Files

| File | Role |
|---|---|
| `openai_benchmark_cases.json` | **V3 candidate cases** — GPT-5.4 output, feeds into Phase 0 validation |
| `sonnet_benchmark_cases.json` | Comparison only — generated by Claude Sonnet under the same prompt to contrast case quality and style across models. Not used in v3. |
