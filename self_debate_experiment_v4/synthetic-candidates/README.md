# synthetic-candidates/

This directory contains raw candidate benchmark cases generated by an external LLM as inputs to the v4 experiment pipeline.

## Generation

Cases were generated using the v4 case generation prompt (`self_debate_experiment_v4/benchmark_case_generation_prompt.md`) executed on **GPT-5.4**. The v4 prompt includes rubric-anchored difficulty definitions, structural requirements for hard cases, a Phase 3 self-test gate, and a difficulty discrimination table — all changes motivated by v3 POST_MORTEM Issue 9 (difficulty labels did not predict baseline performance; rho=-0.069, p=0.68).

## Role in the pipeline

These files are raw LLM output — unvalidated and unverified. They feed into the v4 pipeline starting at Phase 0:

1. **Phase 0** — `validate_cases.py` checks schema compliance and composition targets (category balance, difficulty distribution, defense_wins coverage, must_find population)
2. **Phase 1** — `CASE_VERIFIER` agent reviews each case for flaw authenticity, benchmark quality, and external sourcing of defense_wins cases
3. Output of verification is `benchmark_cases_verified.json`, which is what the experiment actually runs against

The `.json` files in this directory are candidates only. Do not use them directly for scoring or analysis — use `benchmark_cases_verified.json` produced by Phase 1.

## Generator self-critique

The benchmark generator flagged three weaknesses:

1. **~~All mixed-position cases are hard cases.~~** *(Addressed.)* The original 50-case benchmark had all 12 mixed cases at hard difficulty. 4 medium-mixed supplement cases were generated and appended (broken_baseline_051, metric_mismatch_052, hidden_confounding_053, real_world_framing_054), bringing the distribution to 0 easy / 4 medium / 12 hard for mixed-position cases. This enables isolating debate's effect on ambiguous cases from its effect on hard cases.

2. **Domain expertise distribution is concentrated in specific verticals.** Clinical/medical and financial domains account for 7 of the 10 domain-expertise hard cases. A model fine-tuned on healthcare and finance literature would gain disproportionate advantage, reducing the benchmark's ability to measure general reasoning capacity vs. memorized domain facts.

3. **The "trivial baseline competitive" constraint is met but weakly.** The 12 easy cases satisfy the minimum 5 trivial-baseline-competitive requirement, but the easy tier's predicted 0.2 failed dims suggests these cases may be too easy — a keyword-matching baseline could score well, potentially compressing the range at the bottom and making it harder to distinguish marginal baseline improvements from genuine debate-protocol gains.
